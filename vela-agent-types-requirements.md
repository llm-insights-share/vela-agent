# Vela·织帆 三种智能体类型需求规格
### 单体 Agent / 多 Agent 编排 / 工作流型 —— 配置需求与功能实现需求

---

## 0. 概述

本文档从智能体创建向导 IDN-02（智能体类型选择）出发，将三种类型的差异化需求独立抽取成规格文档，分为两层：

- **配置需求（CFG）**：用户在创建向导里能看到、能填写的表单项
- **功能实现需求（IMP）**：平台后台需要实现的运行时能力，支撑对应配置生效

编号规则：`[类型前缀]-[CFG/IMP]-[序号]`，类型前缀为 `SGL`（单体）/`MA`（多Agent编排）/`WF`（工作流型）。优先级定义沿用平台统一标准：**P0** 必须　**P1** 增强　**P2** 长期演进。

三种类型共享 IDN/CAP/SKL/GTW/SEC 等通用模块的基础配置（见第 4 节），本文档只列**类型专属**部分。

---

## 1. 单体 Agent（SGL）

### 1.1 配置需求

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| SGL-CFG-01 | System Prompt 单一编辑器（模板库+变量占位符） | P0 |
| SGL-CFG-02 | ReAct 循环参数：最大迭代次数（默认10） | P0 |
| SGL-CFG-03 | ReAct 循环参数：单步超时时间 | P0 |
| SGL-CFG-04 | 工具调用失败重试策略：重试次数、退避策略 | P1 |
| SGL-CFG-05 | 是否允许连续调用同一工具（防死循环开关） | P1 |
| SGL-CFG-06 | HITL 简化配置：按工具勾选"是否需要人工确认"，非画布节点 | P0 |
| SGL-CFG-07 | 单次调用 Token 上限（成本控制） | P1 |

### 1.2 功能实现需求

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| SGL-IMP-01 | ReActEngine 单节点循环控制器：思考→行动→观察迭代 | P0 |
| SGL-IMP-02 | 循环终止判定：达到最大迭代次数/LLM主动结束/超时三种触发条件 | P0 |
| SGL-IMP-03 | 死循环检测：连续N次调用同一工具且参数相同时强制中断并告警 | P1 |
| SGL-IMP-04 | 工具调用失败自动重试（对接SGL-CFG-04策略配置） | P1 |
| SGL-IMP-05 | 单体 Agent 可作为子图被 MA/WF 类型引用（复用同一套 RunnableAgent 抽象） | P1 |

---

## 2. 多 Agent 编排（MA）

> 配置与实现复杂度最高的类型，核心新增"子智能体管理"与"协调者治理"两层能力

### 2.1 配置需求

**子 Agent 定义**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| MA-CFG-01 | 子 Agent 列表管理：新建专属子 Agent 或引用已发布的单体 Agent | P0 |
| MA-CFG-02 | 每个子 Agent 独立配置角色 Prompt、模型路由、绑定工具/技能/知识库 | P0 |
| MA-CFG-03 | 子 Agent 命名与职责描述（供 Coordinator 分派时识别） | P0 |

**Coordinator 配置**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| MA-CFG-04 | 任务分发策略选择：LLM 自主判断分派 / 规则匹配分派（关键词、意图分类） | P0 |
| MA-CFG-05 | 最大分派轮次配置（防止无限调度） | P0 |
| MA-CFG-06 | 结果整合方式：Coordinator 二次汇总 / 直接拼接子 Agent 输出 | P1 |
| MA-CFG-07 | Coordinator 专用模型路由配置（通常建议强推理模型，与 Specialist 分开配置） | P0 |

**A2A 通信配置**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| MA-CFG-08 | A2A 通信模式：子 Agent 间禁止直连，强制经 Coordinator 中转（默认，便于审计） | P0 |
| MA-CFG-09 | 【可选】允许特定子 Agent 对间直连（需显式声明白名单） | P2 |

**安全与成本配置**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| MA-CFG-10 | 交付前审批 Gate：结果对外输出前强制 HITL（应对路径不固定、可审计性低的问题） | P0 |
| MA-CFG-11 | 任务总 Token 预算上限（防止多轮子 Agent 调用失控超支） | P0 |
| MA-CFG-12 | 单任务最大 A2A 调用次数上限 | P1 |

### 2.2 功能实现需求

**协调调度引擎**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| MA-IMP-01 | CoordinatorEngine：解析用户任务，动态决定调用哪个/哪些子 Agent | P0 |
| MA-IMP-02 | 分派轮次计数器，达到 MA-CFG-05 上限时强制终止并转人工介入 | P0 |
| MA-IMP-03 | 子 Agent 执行结果结构化回传 Coordinator（非自由文本，便于二次整合） | P0 |
| MA-IMP-04 | Coordinator 汇总逻辑：调用独立的汇总 Prompt，将多个子 Agent 输出整合为统一交付物 | P1 |

**A2A 通信层**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| MA-IMP-05 | A2A 协议实现：Coordinator↔Specialist 消息封装、路由、超时控制 | P0 |
| MA-IMP-06 | 通信中转强制校验：拦截子 Agent 间的直接调用尝试（除非命中 MA-CFG-09 白名单） | P0 |
| MA-IMP-07 | 每次 A2A 调用写入审计日志（发起方、接收方、载荷摘要、耗时） | P0 |

**成本与安全治理**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| MA-IMP-08 | 任务级 Token 用量实时累加，超预算（MA-CFG-11）时中断并通知 | P0 |
| MA-IMP-09 | 交付前 HITL Gate 实现：Coordinator 汇总完成后自动创建审批工单，审批通过前结果不下发渠道 | P0 |
| MA-IMP-10 | 子 Agent 故障隔离：单个 Specialist 失败不导致整体任务崩溃，Coordinator 可重试或降级处理 | P1 |
| MA-IMP-11 | 调度轨迹可视化：记录完整的 Coordinator 分派→子Agent执行→回传 链路，供调试与审计回放 | P1 |

### 2.3 参考案例

经济责任审计立项场景：Coordinator + 资料收集 Agent + 风险识别 Agent + 报告撰写 Agent 三个 Specialist 协作，资料收集用轻量模型调用 OA/财务系统 MCP 工具，风险识别绑定审计法规知识库与专用技能包，报告撰写检索历史模板库生成交付物，全流程经 Coordinator 中转、汇总后强制走 HITL 审批 Gate 才对外交付。该案例覆盖了 MA-CFG-01~12、MA-IMP-01~11 的完整调用链路，可作为该类型的标准验收用例。

## 3. 工作流型（WF）

> 可预测性最强、最适合强合规场景的类型，核心是静态可视化编排图

### 3.1 配置需求

**画布与节点**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| WF-CFG-01 | 拖拽式可视化编排画布（对应 ORC-01） | P0 |
| WF-CFG-02 | 节点类型：LLM 节点（绑定 Prompt + 模型） | P0 |
| WF-CFG-03 | 节点类型：工具/技能调用节点 | P0 |
| WF-CFG-04 | 节点类型：条件分支节点（基于上一节点输出做 if/else 路由） | P0 |
| WF-CFG-05 | 节点类型：HITL 审批节点 | P0 |
| WF-CFG-06 | 节点类型：Cron 触发节点（仅工作流型可挂载定时触发入口，对应 ORC-04） | P1 |
| WF-CFG-07 | 节点类型：子图节点（可嵌入一个单体 Agent 作为某环节执行者） | P1 |

**节点级配置**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| WF-CFG-08 | 每个节点独立设置超时时间 | P0 |
| WF-CFG-09 | 每个节点独立设置重试次数与重试间隔 | P1 |
| WF-CFG-10 | 每个节点失败后走向配置：重试 / 跳过继续 / 终止流程并告警 | P0 |
| WF-CFG-11 | 每个 LLM 节点可单独配置模型路由（不同节点用不同模型，如摘要节点用轻量模型、风险判断节点用强推理模型） | P0 |
| WF-CFG-12 | Cron 节点触发的任务必须绑定 HITL/审批流节点，不允许绕过（安全红线，对应 ORC-05） | P0 |

### 3.2 功能实现需求

**编排图编译与校验**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| WF-IMP-01 | LangGraphCompiler：将画布节点定义编译为可执行的 LangGraph 状态机 | P0 |
| WF-IMP-02 | 图连通性校验：检测孤立节点（发布前阻断，见发布流水线Phase 1） | P0 |
| WF-IMP-03 | 环路死锁检测：条件分支若可能形成无退出条件的环路，发布时告警 | P0 |
| WF-IMP-04 | Cron 节点安全校验：未绑定 HITL/审批节点的 Cron 节点禁止发布（对应 WF-CFG-12） | P0 |
| WF-IMP-05 | 节点级模型路由绑定：每个 LLM 节点独立走一次 LLMRouter 路由决策，而非全图共用一个模型配置 | P0 |

**运行时执行引擎**

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| WF-IMP-06 | 状态机执行器：按节点定义顺序执行，条件分支节点评估路由表达式决定下一跳 | P0 |
| WF-IMP-07 | 节点失败处理：按 WF-CFG-10 配置执行重试/跳过/终止逻辑 | P0 |
| WF-IMP-08 | HITL 节点阻塞机制：流程执行到审批节点时挂起，等待人工操作后恢复 | P0 |
| WF-IMP-09 | Cron 触发器：周期性任务到时创建新的流程实例，走完整校验+执行链路（不允许绕过安全闸门） | P1 |
| WF-IMP-10 | 子图节点执行：调用嵌入的单体 Agent 实例（复用 SGL-IMP-01 的 ReActEngine），结果作为该节点输出返回主图 | P1 |
| WF-IMP-11 | 流程执行轨迹全记录：每个节点的输入/输出/耗时/状态写入执行日志，供审计与调试 | P0 |

## 4. 三类型共享需求（简要引用）

以下模块三种类型共用同一套配置表单与实现，不因类型不同而分化，具体条目详见对应模块设计文档：

| 模块 | 共享内容 |
|---|---|
| IDN | 基础档案、可见范围、RBAC权限（IDN-01~03） |
| CAP | 知识库绑定与检索参数（CAP-03~05，但WF可按节点粒度覆盖） |
| SKL | 技能来源、供应链安全审核（SKL-01~09、SEC-05/06/09） |
| GTW | 渠道接入配置（GTW-01~04） |
| SEC | 沙箱隔离、审计日志（SEC-01/02/10，MA/WF在此基础上各自新增专属安全项，见2.1/3.1节） |

---

## 5. 三类型需求量对比

| 维度 | 单体 Agent | 多 Agent 编排 | 工作流型 |
|---|---|---|---|
| 配置需求条目数 | 7 | 12 | 12 |
| 功能实现需求条目数 | 5 | 11 | 11 |
| P0 占比 | 高（结构简单） | 中（安全治理项多为P0） | 高（合规场景要求强） |
| 新增专属安全机制 | 死循环检测 | 交付前HITL Gate、Token预算熔断 | Cron+HITL绑定红线、图连通性校验 |
| 可复用单体Agent能力 | — | 子Agent可复用SGL | 子图节点可复用SGL |
